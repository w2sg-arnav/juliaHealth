Part 4

Description: Build a simple dense network and find the optimal learning rate.

First, I define a sequential chain of layers to be called with their associated activation functions:

simple_dense_network = Chain(
    #=
        Input dimension of 28^2 comes from each MNIST image
        being flattened to a vector of length 784.
        An output parameter space of 300 was chosen to improve
        training and relu activation was picked for this layer.
    =#
    Dense(28^2 => 300, relu),

    #=
        Input dimension of 300 comes from the prior layer.
        An output parameter space of 100 was chosen to improve
        training and relu activation was picked for this layer.
    =#
    Dense(300 => 100, relu),

    #=
        Input dimension of 100 comes from the prior layer.
        An output parameter space of 10 was selected to match
        to the given targets for the MNIST dataset (i.e. what
        are the prediction values for each image).
    =#
    Dense(100 => 10),

    #=
        Apply a softmax function to the previous layer and
        reports the probability for each value.
    =#
    softmax
)

Then I define some helper functions:

#=
    Convenience function to have data altogether in one 
    place for training. Specifically encoded for MNIST
    and to support batch training.
=#
function simple_loader(data::MNIST; batchsize::Int=64)
    x2dim = reshape(data.features, 28^2, :)
    yhot = Flux.onehotbatch(data.targets, 0:9)
    Flux.DataLoader((x2dim, yhot); batchsize, shuffle=true)
end

#=
    Calculates accuracy for a given model
=#
function simple_accuracy(model, data::MNIST=test_data)
    (x, y) = only(simple_loader(data; batchsize=length(data)))
    y_hat = model(x)
    iscorrect = Flux.onecold(y_hat) .== Flux.onecold(y)
    acc = round(100 * mean(iscorrect); digits=2)
end

Then I define the stochastic gradient descent optimizer I want to use during training optimization:

learning_rate = 1e-3
optimizer = Descent

simple_optim = Flux.setup(
    optimizer(learning_rate), 
    simple_dense_network
);

Batch training data:

train_loader = simple_loader(train_data, batchsize = 256)

Train the model and report loss (cross entropy), training accuracy, and test accuracy:

epochs = 100
global losses = []
global train_accs = []
global test_accs = []

for epoch in 1:epochs
    model_loss = 0.0
    for (x, y) in train_loader
        curr_loss, gradients = Flux.withgradient(m -> Flux.crossentropy(m(x), y), simple_dense_network)
        Flux.update!(simple_optim, simple_dense_network, gradients[1])
        model_loss += curr_loss / length(train_loader)

    push!(train_accs, simple_accuracy(simple_dense_network, train_data))
    push!(test_accs, simple_accuracy(simple_dense_network, test_data))
    push!(losses, model_loss)
end

Addendum

Concerning problem $2$, this problem proved extremely challenging for a variety of reasons.

The first reason was that the design of a neural network for determining the extent of Alzheimer's in someone based on their MRI scan is extremely challegning as a problem itself. This is still an ongoing problem within much of biomedical signal processing and medical image processing research. It can be called a hard problem as the simple application of a neural network will not solve this problem space.

Second, the way to evaluate this problem can vary dramatically depending on the architecture chosen for the neural network and the architecture for the computer the neural network is running on. I consistently received a test accuracy of $\approx 25\%$ for this problem when using only a dense neural network this caused some alarm in me. I conferred with a classmate, Maria Georgieva, about this problem and she too saw similar behavior where accuracy did not seem to vary greatly even if changing the number of dense layers or neurons used in said layers. Finally, to clarify my confusion, I conferred with a mathematics PhD student acquaintance of mine at the Universite de Toulon a La Garde, Yolhan Mannes.

Yolhan and I came up with a very different model (convolutional model) that did improve loss and accuracy but went beyond the scope of this problem:
